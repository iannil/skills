## 可信度分级 / Confidence Tiers

每一条侦察资产必须落入以下三层之一（定义同 `SKILL.md` 原则二）。分级是本报告的**交付本质**。

| 层级 / Tier | 含义 / Meaning | 构建处理 / Build Handling |
|:-----------|:--------------|:------------------------|
| **明示 / Stated** | 材料白纸黑字写着：菜单名、表头列、按钮文案、筛选项、状态标签、统计数字、写出来的校验文案。可直接采信其存在。 | 精确落地为字段/操作/状态/报表。 |
| **推断 / Inferred** | 从明示证据 + 业务常识反推：字段类型、关系基数、状态转移、软删除语义、未展开枚举。行为等价，非确证。 | 按推断实现，标注"待用户确认"；用户确认后升级为明示。 |
| **缺口 / Gap** | 材料完全没提：后端算法、精确权限矩阵、字段级校验（无文案）、支付/税务/物流等外部集成内部、定时任务。 | 进下方缺口清单，向用户追问；构建时 mock/占位，**禁止臆造**。 |

> **标注纪律**：一项一签，拿不准就下沉（`推断` 优先于 `明示`，涉及后端内部一律 `缺口`）。
